Token Taxonomy Artifact

Base-Token

All tokens will share a simple set of common properties and provide a base set of expectations when browsing a collection of tokens on a network and also must be provided when creating a new token class.

All Tokens are Constructable, a behavior that is not formally defined, but implies that every Token has an ability to create or construct a clone of itself. This behavior is active when a token is operating as a template and not a class and can receive a request to create a clone or token class of itself. This control is generically called a constructor.

Base Token Properties:

NameA common well understood name that represents the Token Class. All instances, or tokens, within this class will be referred to by their class name.SymbolA common well understood name that represents the Token Class. All instances, or tokens, within this class will be referred to by their class name.OwnerA reference to the owner of the token class or instance which can be a blockchain address, public key or other unique identifier.QuantityCan represent the initial quantity created or the total minted or issued.DecimalsA number of decimal places a single token can be subdivided into. A typical fiat currency has a value of 2, i.e. $100.53. A value of 0 means that subdivision is not supported and a whole token is the smallest unit of the token that can be owned. PropertiesNon-behavioral properties as a list of simple `name, value` pairs that can be generically implemented.

These represent non-behavioral properties, with the exception of Decimals which is used by the Subdividable or Non-Subdividable behavior where any value greater than 0 makes the token subdividable.
