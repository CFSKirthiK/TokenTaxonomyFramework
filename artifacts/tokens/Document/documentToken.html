Token Taxonomy Artifact

Document Token

Alternate names: File, Invoice, Bill of Ladling, PDF

Used to represent a document that may be a scanned or PDF printed document. It records the document hash to check for tampering, a file path to fetch the file from storage as well as the ability to be encumbered.

For example: you may choose to create an invoice token from an invoice document. You can then allow another token or contract representing a loan or proof of financing to encumber the document establishing a link between the two.

Token Properties

Hash algorithm used to generate the document hash, a string value for then name used like SHA256 or SHA3.

Document Hash is a hash value for the document. This value is useful to allow to check if the document was tampered since creating the token.

Location where the actual document can be retrieved from to compare the document hash.

This is a data structure used to record the information about the party that owns the encumber.

Token Behaviors:

Transferable - This token can transfer ownership. This is restricted if encumbered.

Singleton - There will only be a single token.

Non-subdividable - The single token is the smallest ownable unit.

Burnable - The token can be retired or made permanently inactive. This is restricted if encumbered.

Encumberable - The document can support being encumbered by another party and restricting certain behaviors.

Behaviors

Implementation should include a retrieve or fetch method that allows for the properties of the document to be queried and updated if needed.
