Burnable Behavior

A token class that implements this behavior will support the burning or decommissioning of token instances of the class. This does not delete a token, but rather places it in a permanent non-use state. Burning is a one way operation and cannot be reversed.

This behavior is Delegable. If the token definition is Delegable, BurnFrom will be available.

Behavior Invocations:

BurnRequestA request to burn a token instance(s) in the class by the owner of the token instance(s). Optional Quantity field in the request.BurnResponseA confirmation receipt or error may be returned to the invoker based on the outcome of the burn request
BurnFromRequestIf DelegableA request to burn token instance(s) in the class by a party or account that has allowance to do so. Requires a From and Quantity fields in the request.BurnFromResponseIf DelegableA confirmation receipt or error may be returned to the invoker based on the outcome of the BurnFrom request.

Comments

Should Allowance be defined in a separate behavior, like Delegable?

Yes, because it can be used by both transfer and burn behaviors and if applied in conjunction with this behavior it will include BurnFrom.
